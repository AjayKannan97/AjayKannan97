### A bit about me

I'm a software engineer based in **Santa Clara, CA**, currently at **Pacific Gas & Electric**, where I build the data platforms and backend services that keep large utility systems honest — pipelines, validation, observability, and a bit of ML where it earns its keep.

Before PG&E, I did my **M.S. in Computer Science at Arizona State University**, where I got hooked on deep learning and few-shot methods. Earlier stops include **Microsoft** (FarmBeats and Cortana), **Sigtuple** (medical image analytics), and **Solarillion Foundation**, where I led an IoT research group through college.

These days I spend my evenings tinkering with **LLMs, RAG, and on-device inference** — partly because it's fun, partly because the field is moving too fast to watch from the sidelines. If any of that sounds interesting, I'd love to chat.

🌐 More on my **[portfolio](https://www.ajaykannan.site)** · ✍️ Occasionally I write on **[Medium](https://medium.com/@ajay_k)**

### What I'm good at, honestly

- 🏗️ **Building reliable backend systems** — designing data pipelines, REST/GraphQL APIs, and observability into things that need to *stay up*
- 🧪 **Applied ML** — using PyTorch and the usual ML stack on real problems (anomaly detection, image analytics, few-shot learning)
- 🤖 **Generative AI, with the right level of humility** — I've shipped LangChain + RAG prototypes and on-device LLM assistants, and I'm continuously learning the production side (Bedrock, vector DBs, evals)
- 🤝 **Working with people** — design reviews, code reviews, mentoring, and writing the kind of docs other engineers actually read

I try to be the engineer who asks "why" before "how," who finishes things, and who leaves systems a little better than I found them.
